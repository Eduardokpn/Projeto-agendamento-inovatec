# 🎯 Visão Geral do Projeto: Desafio Técnico InovaTec 26.1

## Informações Básicas
* **Organização:** InovaTec / CPE USP
* **Tempo Estimado:** 3 semanas
* **Complexidade:** Média
* **Formato:** Trabalho em trio

## O Problema (Cenário Atual)
O departamento Pedagógico do cursinho oferece o "Planejamento Estratégico", onde alunos agendam atendimentos individuais para orientação de estudos. 
Hoje, isso é feito através de uma **Planilha (Excel/Google Sheets)** que funciona como um sistema de agendamento manual.
* **A dor:** A planilha é confusa, sujeita a erros (alunos editando onde não devem), possui muitos campos vazios e a experiência de uso (UX) tanto para o aluno quanto para o membro da equipe não é otimizada.

## O Objetivo do Desafio
Desenvolver uma solução tecnológica para resolver (total ou parcialmente) os problemas de agendamento. 
* **Liberdade Técnica:** A equipe tem *total liberdade* para não usar planilhas. Pode-se construir a solução com qualquer ferramenta, tecnologia ou abordagem (ex: Bubble.io, Glide, React, etc.).
* **Foco:** Criar um sistema eficiente, prático e escalável.

## Etapas Obrigatórias do Desenvolvimento
1. **Planejamento:** Entendimento da dor, discussão em trio e definição da arquitetura da solução.
2. **Mão na Massa:** Desenvolvimento real do sistema (banco de dados, telas, automações).
3. **Documentação:** Registro das decisões tomadas e manual de uso, que deverá ser feito na plataforma **Notion** (no workspace disponibilizado para o grupo).

## Critério de Entrega
A entrega oficial é a própria **Documentação no Notion**. Ela deve conter a explicação de como o sistema funciona e os links de acesso à aplicação desenvolvida (ou link do repositório no GitHub, se houver código puro).
